Hero Kitchen App

Technicians clock in with GPS-verified photo capture, file structured pest-control service reports with dual signatures, and submit expense requests. Admins track employees live on a map, approve leaves and disputes, and manage schedules. Built offline-first with a full background sync service and a WEA-style 14-hour work-duration safety alert.

Offline-first clock-ins — GPS position, client, and photo are queued in a JSON-backed SharedPreferences store when offline; a background sync service submits them all when connectivity returns.
WEA-style 14-hour work safety alert — AlarmManager.setAlarmClock schedules an exact wake-up; on fire, a full-screen activity launches through a full-screen notification intent, with VibrationAttributes.USAGE_ALARM overriding DND and silent mode.

SmartGeocoder with spatial cache — Reverse-geocoded addresses cached in SharedPreferences, returned when a new request is within 500m of a prior point.
Role-based UI (5 roles) — Technician, BackOffice, HouseKeep, Admin, SuperAdmin each get a tailored bottom-nav (menu items dynamically removed) and a different default landing fragment.
Complete CPAR workflow — Pest-specific service reports (Cockroach, Fly, Mosquito, Rodent, Cat, Others) with per-report preview activities and dual client+technician signature capture before submission.
Revolving Funds — Auto-generated reference numbers, TIN auto-complete, VAT/Non-VAT auto-computation, photo attachments, and a PHP/MySQL backend with three linked tables for full audit trails.
Forced update gate — Version check against a remote endpoint blocks the app until the user installs a new APK via DownloadManager + PackageInstaller.
Boot-persistent services — BootReceiver + ServiceRestartReceiver resume location tracking automatically after device restart.
OTP password recovery — Mobile-number-first recovery via Firebase Phone Auth, bridged to email-account password reset through the Firebase Admin REST API.

Mobile App Worship Team Tool

Mic Test

A native Android companion app for worship teams. Musicians browse a shared song library with chord charts, transpose on the fly across all 12 keys, and import lyrics from photos using on-device OCR that preserves the original chord-over-lyric alignment. Leaders plan services across multiple church satellites with per-location song leader rosters and shared song lists.

Context-aware chord highlighting — A regex parser distinguishes chord lines from lyric lines and validates matches against surrounding text so "A" in "Amazing" isn't mistaken for a chord.
Full chromatic transposer — Shifts lyrics across all 12 keys with recursive slash-chord handling (D/F# transposes both root and bass) and preserves chord quality (m, maj, sus, 7, 9, 11…).
OCR song import — ML Kit Text Recognition reads lyrics from a photo; per-word bounding-box coordinates are used to insert spacing so chord letters stay aligned over the right syllables.
Multi-satellite service planning — Different song-leader rosters per church location (WOH Acacia, Arko, Pandi), with custom guest entries and live add/remove of songs from a scheduled service.
Deterministic "Verse of the Day" — Date-seeded RNG ensures every user sees the same verse each day; fetched once and cached in SharedPreferences to avoid repeat API calls.
Role-based access control — Five roles (Admin, Song Leader, Musician, Media, Member) with color-coded badges and gated UI (add/edit/delete appears only for authorized roles).
Dual backend architecture — Firebase Authentication + Realtime Database for user/auth state; Supabase (PostgREST via Retrofit) for songs, services, and announcements.
Selective Gson serialization — @Expose annotations keep auto-generated Postgres fields (id, created_at, updated_at) out of POST bodies while still reading them on responses.
